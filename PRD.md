# 智能待办 - 产品需求文档

## 1. 项目概述

「智能待办」是一款面向个人的效率工具，集成待办事项管理、学习计划跟踪和备忘录管理三大功能模块，并通过 AI 对话实现智能化的数据创建、修改与推荐。所有数据存储在浏览器本地，无需后端服务器。

### 1.1 核心特性

- 三位一体：待办事项 + 学习计划 + 备忘录统一管理
- AI 驱动：通过自然语言对话直接创建、修改、管理所有模块数据
- 上下文感知：AI 自动获取当前所有模块状态作为对话上下文
- 离线可用：数据存储在浏览器本地，核心功能无需网络

### 1.2 技术决策

| 维度 | 选型 |
|---|---|
| 应用架构 | SPA 单页应用 |
| 前端框架 | React 18 + TypeScript |
| UI 组件库 | Ant Design 5 + Ant Design Pro Layout |
| 路由方案 | React Router v6 |
| 状态管理 | zustand |
| 本地存储 | IndexedDB (dexie.js) |
| AI 服务 | 智谱 ChatGLM API（Function Calling） |
| 日期处理 | dayjs |
| 构建工具 | Vite |

## 2. 首页设计

首页作为 Dashboard 聚合所有信息，采用 Ant Design Pro 的 ProLayout 布局，分为四大区域：

### 2.1 装饰展示区（顶部 Header）

放置动态小组件，提供视觉丰富度：
- 实时时钟
- 天气信息（可接入免费天气 API）
- 每日名言/格言轮播
- 轻量互动小游戏（待定具体内容）

### 2.2 功能导航区（左侧边栏）

三个模块的入口：
- 待办事项
- 学习计划
- 备忘录

当前活跃模块高亮显示。

### 2.3 进度展示区（中央主区域 - Dashboard 视图）

聚合展示各模块的进度概览，展示顺序：
1. **备忘录摘要** — 最近备忘录列表，支持快速查看和操作
2. **待办事项概览** — 今日待办、逾期待办、完成率统计
3. **学习计划进度** — 各计划完成进度条、今日打卡状态

### 2.4 AI 对话区（右下角浮动窗口）

- 常驻浮动按钮，点击展开对话窗口
- 统一入口，可处理所有模块事务
- 支持收起/展开，不影响正常使用
- 详见 [第 6 节 AI 能力](#6-ai-能力)

## 3. 模块一：待办事项

### 3.1 基础功能

- **CRUD**：创建、查看、编辑、删除待办事项
- **字段定义**：
  - 标题（必填）
  - 描述（选填）
  - 优先级（1-5 级，1 最高）
  - 截止日期（选填）
  - 标签/分类（选填）
  - 完成状态（布尔值）
  - 创建时间（自动）
- **筛选与排序**：按优先级、截止日期、完成状态筛选；支持自定义排序
- **到期提醒**：使用浏览器 Notification API 在截止时间前提醒
- **批量操作**：支持批量标记完成、批量删除

### 3.2 AI 能力

- **自然语言创建**：用户描述任务，AI 解析并创建待办事项（自动设置优先级、日期）
- **智能排序**：AI 根据紧急程度、依赖关系对待办事项重新排序
- **流程推荐**：AI 生成任务执行的全流程建议（步骤拆解、时间分配）
- **自动修改**：AI 可直接修改已有待办事项的内容、优先级、截止日期

---

## 4. 模块二：学习计划

### 4.1 基础功能

- **计划管理**：创建、编辑、删除学习计划
- **任务列表**：每个计划包含多个学习任务，支持分点勾选
- **打卡功能**：记录每日学习打卡，展示连续打卡天数
- **进度追踪**：可视化展示每个计划的完成进度（进度条）
- **学习记录**：记录每次学习的时长和内容笔记
- **字段定义**：
  - 计划标题、描述
  - 任务列表（嵌套结构，支持子任务）
  - 每个任务：标题、完成状态、学习资源链接
  - 打卡记录：日期数组
  - 创建/更新时间

### 4.2 AI 能力

- **对话生成计划**：用户描述学习目标，AI 生成结构化的学习计划（含任务拆解）
- **学习路线**：AI 生成完整的学习路线图，按阶段划分
- **资源推荐**：根据学习目标和当前进度推荐学习资源
- **动态调整**：根据完成情况，AI 可调整计划难度和节奏
- **自动修改**：AI 可直接修改学习计划的任务列表和内容

---

## 5. 模块三：备忘录

### 5.1 基础功能

- **CRUD**：创建、查看、编辑、删除备忘录
- **字段定义**：
  - 标题（必填）
  - 内容（支持 Markdown 格式）
  - 标签（选填）
  - 创建/更新时间
- **搜索**：支持按标题和内容全文搜索
- **转待办**：将备忘录内容一键转为待办事项

### 5.2 AI 能力

- **智能管理**：AI 可对备忘录进行分类、摘要、标签建议
- **转待办**：AI 从备忘录中提取待办事项并创建
- **自动创建**：通过对话让 AI 直接创建备忘录
- **自动修改**：AI 可修改已有备忘录内容

## 6. AI 能力

### 6.1 交互模式

AI 对话是统一入口，位于首页右下角浮动窗口。核心原则：**AI 不仅是建议者，更是执行者。**

### 6.2 上下文自动注入

打开 AI 对话时，系统自动将以下数据作为上下文传入：
- 所有待办事项（标题、优先级、截止日期、完成状态）
- 所有学习计划（标题、任务列表、完成状态、打卡记录）
- 所有备忘录（标题、内容摘要、标签）

上下文传入策略：
- 数据量正常时（< 50 条记录）：传入完整数据
- 数据量大时（> 50 条记录）：仅传入摘要（如未完成事项、最近7天记录）

### 6.3 结构化操作

AI 通过智谱 API 的 Function Calling 能力执行数据操作。支持的操作：

| 操作 | 说明 | 目标 |
|---|---|---|
| `add_todo` | 创建待办事项 | 待办模块 |
| `update_todo` | 修改待办事项 | 待办模块 |
| `delete_todo` | 删除待办事项 | 待办模块 |
| `add_plan` | 创建学习计划 | 学习计划模块 |
| `update_plan` | 修改学习计划 | 学习计划模块 |
| `delete_plan` | 删除学习计划 | 学习计划模块 |
| `add_memo` | 创建备忘录 | 备忘录模块 |
| `update_memo` | 修改备忘录 | 备忘录模块 |
| `delete_memo` | 删除备忘录 | 备忘录模块 |

### 6.4 确认机制

AI 的所有写操作（创建、修改、删除）在执行前需展示变更预览：
1. AI 返回操作指令 + 文本说明
2. 前端在对话窗口中展示变更预览卡片
3. 用户点击「确认执行」后，数据变更生效
4. 变更结果反馈到对话和对应模块 UI

### 6.5 典型对话场景

| 用户输入 | AI 行为 |
|---|---|
| "帮我安排明天的任务：上午开会，下午写报告" | 创建 2 个待办事项，自动设置日期和优先级 |
| "学习计划里的 React 基础太简单了，换成 advanced 的" | 修改学习计划任务列表，推荐新资源 |
| "我这周还有很多没完成" | 读取未完成事项，建议重新排期并可一键执行 |
| "把备忘录'项目想法'转成待办" | 从备忘录创建待办事项 |
| "我想学 TypeScript，帮我做个学习计划" | 生成完整学习计划（含任务拆解和资源推荐） |
| "今天心情好，推荐一句名言" | 返回名言文本（无数据操作） |

## 7. 非功能性需求

- **性能**：首屏加载 < 2s，模块切换 < 300ms
- **响应式**：支持桌面端（主要）和移动端（基本可用）
- **数据持久化**：所有数据存储在 IndexedDB，关闭浏览器不丢失
- **API Key 管理**：用户在设置页输入智谱 API Key，存储在 localStorage
- **部署**：构建后可部署到任意静态托管服务（Vercel、GitHub Pages 等）
