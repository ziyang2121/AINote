# 智能待办 - 分阶段开发计划

## 阶段总览

| 阶段 | 名称 | 目标 | 交付物 |
|---|---|---|---|
| P0 | 项目脚手架与基础架构 | 搭建项目骨架，跑通技术选型 | 可运行的基础项目框架 |
| P1 | 首页 Dashboard | 完成首页四大区域的 UI 搭建 | 可交互的首页 |
| P2 | 待办事项模块 | 完整的待办 CRUD + 数据持久化 | 可用的待办管理功能 |
| P3 | 学习计划模块 | 计划管理 + 打卡 + 进度追踪 | 可用的学习计划功能 |
| P4 | 备忘录模块 | 备忘录管理 + Markdown 支持 | 可用的备忘录功能 |
| P5 | AI 对话核心 | 对话窗口 + Function Calling + 上下文注入 | AI 可创建/修改所有模块数据 |
| P6 | 体验优化与收尾 | 响应式、通知提醒、性能优化 | 完整可用的产品 |

---

## P0 - 项目脚手架与基础架构

### 目标
搭建项目基础骨架，配置所有工具链，确保开发环境就绪。

### 任务清单

- [ ] 使用 Vite 创建 React + TypeScript 项目
- [ ] 安装核心依赖：
  - `antd` + `@ant-design/pro-layout`
  - `react-router-dom`
  - `zustand`
  - `dexie`（IndexedDB）
  - `dayjs`
- [ ] 配置项目目录结构（pages / components / services / stores / types / utils）
- [ ] 配置路由：
  - `/` → Dashboard
  - `/todo` → 待办事项
  - `/learning` → 学习计划
  - `/memo` → 备忘录
- [ ] 配置 Ant Design Pro Layout（侧边栏 + 内容区）
- [ ] 定义 TypeScript 类型（todo.ts / learning.ts / memo.ts / ai.ts）
- [ ] 初始化 IndexedDB（dexie.js schema 定义）
- [ ] 配置 ESLint + Prettier（可选）

### 验收标准
- `npm run dev` 启动无报错
- 四个路由可正常切换
- Ant Design Pro Layout 正确渲染（侧边栏 + 内容区）

## P5 - AI 对话核心

### 目标
实现 AI 对话功能，支持上下文注入和 Function Calling，让 AI 可以直接操作所有模块数据。

### 任务清单

- [ ] **设置页**
  - [ ] API Key 输入表单（存储在 localStorage）
  - [ ] API 连通性测试按钮
  - [ ] 模型选择（glm-4-flash 等）
- [ ] **上下文构建**
  - [ ] 实现 `contextBuilder.ts`
  - [ ] 从 IndexedDB 读取所有模块数据
  - [ ] 格式化为 system prompt
  - [ ] 大数据量时启用摘要策略
- [ ] **Function Calling**
  - [ ] 定义 9 个工具函数的 schema
  - [ ] 实现 AI 响应解析（文本 + tool_calls）
  - [ ] 实现操作执行器（ActionExecutor）
    - [ ] add_todo / update_todo / delete_todo
    - [ ] add_plan / update_plan / delete_plan
    - [ ] add_memo / update_memo / delete_memo
- [ ] **确认机制**
  - [ ] ActionPreview 组件（显示操作预览）
  - [ ] 确认/取消按钮
  - [ ] 执行后反馈到对话和 UI
- [ ] **对话管理**
  - [ ] 对话历史持久化（IndexedDB）
  - [ ] 新建对话 / 清空对话
  - [ ] 流式输出（SSE，如果智谱 API 支持）
- [ ] **AI 对话窗口增强**
  - [ ] 打开时自动注入最新上下文
  - [ ] 加载状态 / 错误处理
  - [ ] 对话中的 Markdown 渲染

### 验收标准
- 输入 API Key 后可正常对话
- "帮我创建一个待办：明天买菜" → AI 创建待办事项，确认后生效
- "把学习计划的第一个任务标为完成" → AI 修改学习计划，确认后生效
- 操作确认卡片正确展示操作内容和预览
- 对话历史刷新后保留

## 依赖关系

```
P0 (脚手架)
  ↓
P1 (首页) ←── 无数据依赖，可用 Mock
  ↓
P2 (待办) ── P3 (学习计划) ── P4 (备忘录)    ← 三个模块互相独立，可并行开发
  ↓                ↓                ↓
  └────────────────┼────────────────┘
                   ↓
              P5 (AI 对话核心)              ← 依赖三个模块的数据层
                   ↓
              P6 (体验优化)                  ← 最后阶段
```

**P2、P3、P4 三个模块相互独立，可以并行开发或按任意顺序开发。**
